1. Setup & Installation

Step 0: Install TypeScript

First, you need to install the TypeScript compiler (`tsc`) globally on your computer. This allows you to run the compiler from any directory in your terminal.

npm install -g typescript

Step 1: Create Your First File

Navigate to your project directory (e.g., Week 4/Day 4) in the command line and create a new file. It's crucial to use the `.ts` extension for TypeScript files.

# In your terminal
touch typescriptBasics.ts

2. Core Concepts: Type Safety & Compilation

This section covers the two most fundamental concepts that make TypeScript powerful: its ability to enforce types and the required compilation step to turn TypeScript code into JavaScript that can be run anywhere.

Step 2: Experience Type Safety

TypeScript's main feature is type safety. Let's see it in action. In the editor below, we declare that the `age` variable must be a number. Try to assign a string to it to see the error message appear.

Here is the code to paste into your `typescriptBasics.ts` file. Your code editor (like VS Code) will show a similar error.

let age: number;
age = 30;
age = "thirty"; // This line will cause an error

Step 3: Compiling to JavaScript

Browsers and Node.js don't understand TypeScript directly. You must compile your `.ts` file into a plain `.js` file using the `tsc` command.

tsc typescriptBasics.ts

This command creates a `typescriptBasics.js` file. If your `.ts` file has errors, the compiler will report them and may not create the `.js` file.

Key Concept: The compiler is the bridge between TypeScript and JavaScript. It checks for type errors and translates your code into a runnable format.

3. Writing TypeScript Code

Now let's explore the practical side of writing TypeScript, including how it cleverly infers types for you and the basic types you'll use most often.

Step 4: Type Inference

TypeScript is smart. If you declare and initialize a variable in one step, it will automatically infer the type. You don't have to write it out every time.

let greeting = "Hello, world!"; // TypeScript infers type 'string'
// greeting = 100; // This would cause an error

Step 5: Basic Types (Boolean, Arrays)

Here are examples of other common types. Note the syntax for an array of numbers: `number[]`.

// Boolean
let isLoggedIn: boolean = false;

// Array of numbers
let scores: number[] = [10, 23, 45];

console.log("Is logged in:", isLoggedIn);
console.log("Scores:", scores);

Step 6: The `any` Type

Sometimes you need to opt-out of type checking. The `any` type lets a variable hold any type of value. Use this sparingly, as it removes the safety net TypeScript provides.

let dynamicValue: any;

dynamicValue = 100;
console.log(dynamicValue);

dynamicValue = "Now I'm a string!";
console.log(dynamicValue);

Key Concept: Prefer explicit types or type inference over `any`. Using `any` should be a deliberate choice when dealing with dynamic data or third-party libraries that aren't typed.

4. Functions & Advanced Types

Let's level up by learning how to type functions to make them more robust and how to use union types to allow variables to hold more than one type of value.

Step 7: Typing Functions

You can add types to function parameters and specify the function's return type. This makes your functions predictable and self-documenting.

function add(a: number, b: number): number {
  return a + b;
}

let sum = add(5, 10);
console.log("The sum is:", sum);

Step 8: Union Types

A union type allows a variable to hold a value of several different, specified types. Use the pipe (`|`) symbol to create a union.

let userId: string | number;

userId = 12345;
console.log("User ID:", userId);

userId = "user-abc-123";
console.log("User ID:", userId);
